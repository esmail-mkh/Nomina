## ⬇️ دانلود

از [**آخرین ریلیز**](https://github.com/esmail-mkh/nomina/releases/latest) یک نسخه‌ی آماده‌ی نصب دانلود کنید — بدون نیاز به نصب پایتون.

| پلتفرم | فایل | توضیح |
|---|---|---|
| 🪟 **ویندوز (x64)** | `Nomina-vX.Y.Z-windows-x64.zip`  | از حالت فشرده خارج کن و `Nomina.exe` را اجرا کن |
| 🐧 **لینوکس (x64)** | `Nomina-vX.Y.Z-linux-x64.tar.gz` | با `tar -xzf` باز کن و `./Nomina/Nomina` را اجرا کن |

هر ریلیز همراه با فایل `SHA256SUMS.txt` منتشر می‌شود تا بتوانید صحت دانلود را بررسی کنید.

## ✨ معرفی

**نومینا** کار خسته‌کننده‌ی تغییر نام ده‌ها — یا هزاران — فایل را به یک تجربه‌ی روان و بصری تبدیل می‌کند. یک پوشه را رها (Drop) کنید، قانون‌ها را کنار هم بچینید (پیشوند، پسوند، جایگزینی، برش، تبدیل، شماره‌گذاری) و ببینید چطور نام‌های جدید به‌صورت زنده آپدیت می‌شوند — تازه پیش از اینکه حتی یک فایل روی دیسک عوض شود.

> با وسواس ساخته شده برای وضوح و سرعت. بدون چیزهای اضافه، بدون ردیابی، بدون سورپرایز.

## 🧩 موتور قانون‌ها

با ترکیب این قانون‌ها می‌توانید زنجیره‌ی تغییر نام پیچیده بسازید:

- **پیشوند / پسوند** — افزودن متن قبل یا بعد از نام
- **جایگزینی** — Find & Replace با امکان استفاده از Regex
- **حذف** — پاک کردن بخش‌های اضافی از نام
- **برش (Slice)** — نگه‌داشتن بخشی از نام اصلی بر اساس ایندکس
- **تبدیل حروف** — UPPERCASE، lowercase، Title Case، camelCase، kebab-case، snake_case
- **شماره‌گذاری** — عددگذاری ترتیبی با شروع، گام و تعداد صفر دلخواه
- **پسوند فایل** — تغییر یا یکدست‌سازی پسوند فایل‌ها

هر قانون **مستقل قابل فعال/غیرفعال** است و می‌توانید روی **همه‌ی آیتم‌ها** یا فقط **آیتم‌های انتخاب‌شده** اعمالش کنید.

## 📦 نصب

### پیش‌نیازها

- **Python 3.10 یا بالاتر**
- **ویندوز 10/11** (Edge WebView2)، **لینوکس** (GTK 3 + WebKit2GTK 4.1)، یا **macOS 11+**

### Setup

```bash
# 1. Clone the repository
git clone https://github.com/esmail-mkh/nomina.git
cd nomina

# 2. (Recommended) Create a virtual environment
python -m venv .venv
# Windows:
.venv\Scripts\activate
# Linux / macOS:
source .venv/bin/activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Run the app
python main.py
```

> 💡 **نکته:** در اولین اجرا، نومینا پوشه‌ی تنظیماتش را اینجا می‌سازد:  
> `Documents/EMKH_Apps/Nomina/` (در لینوکس/مک از `~/Documents/...` استفاده کنید) — برای ذخیره‌ی پریست‌ها و تنظیمات.

## 🏗️ بیلد از سورس

با PyInstaller می‌توانید یک اپلیکیشن مستقل و قابل توزیع برای پلتفرم خودتان بسازید.

```bash
# Install build tooling
pip install pyinstaller cairosvg pillow

# Generate platform-specific icons from build/icon.svg
python build/make_icons.py

# Build (onedir) — output goes to dist/Nomina/  (or dist/Nomina.app on macOS)
pyinstaller build/nomina.spec --noconfirm --clean
```

برای کاهش حجم خروجی، **UPX** نصب کنید و `--upx-dir /path/to/upx` را اضافه کنید (فقط روی ویندوز و لینوکس — مک به‌صورت خودکار رد می‌شود). فایل spec از قبل لیست استثناهای ایمن را دارد.

### 🤖 ریلیز خودکار (GitHub Actions)

این پروژه یک workflow دستی در [`.github/workflows/release.yml`](.github/workflows/release.yml) دارد که:

۱. ✅ هنگام اجرا، **شماره‌ی نسخه** را از شما می‌پرسد  
۲. 🛠️ به‌صورت موازی روی **Windows**، **Linux** و **macOS** بیلد می‌کند  
۳. 🎨 آیکون‌های مخصوص هر پلتفرم را از `build/icon.svg` می‌سازد  
۴. 📦 با **آخرین نسخه‌ی UPX** (به‌طور خودکار از مخزن رسمی دانلود می‌شود) فشرده می‌کند  
۵. 🚀 یک **GitHub Release** با هر سه آرشیو + `SHA256SUMS.txt` منتشر می‌کند  

برای انتشار یک ریلیز: **Actions → Build & Release → Run workflow → وارد کردن شماره نسخه → اجرا.**

## 🎬 استفاده

۱. یک پوشه را **بارگذاری** کنید — روی *Browse* بزنید، مسیر را Paste کنید، یا پوشه را روی پنجره رها کنید.  
۲. (اختیاری) **فیلتر** کنید تا فقط آیتم‌های موردنظرتان دیده شوند.  
۳. (اختیاری) آیتم‌هایی را **انتخاب** کنید — اگر چیزی تیک نخورد، قانون‌ها روی همه اعمال می‌شوند.  
۴. **قانون‌ها** را تنظیم کنید — Prefix / Suffix / Replace / Numbering / Transform و... را فعال کنید.  
۵. **پیش‌نمایش** ببینید — ستون *نام جدید* به‌صورت زنده آپدیت می‌شود.  
۶. روی **Validate** بزنید تا نام‌های نامعتبر یا تداخل‌ها مشخص شوند.  
۷. روی **Rename** بزنید تا تغییرات روی دیسک اعمال شوند.
